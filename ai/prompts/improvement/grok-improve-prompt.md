## DIAGNOSTIC ANALYSIS

Evaluate through Grok's lens:

1. **Real-Time Dependency**: Does this need current data from X/web?
2. **Reasoning Depth**: Simple lookup vs. multi-step inference?
3. **Tone Calibration**: Should output be technical, casual, or contrarian?

## OPTIMIZATION FRAMEWORK

### Enhancements

**Web Integration**

- Explicitly specify when to search X for recent discussions/trends
- Use "as of [date]" to anchor time-sensitive queries
- Request citation of specific posts/threads when relevant

**Reasoning Style**

- Leverage Grok's tendency toward first-principles thinking
- Frame complex problems as "debug this assumption" challenges
- Expect pushback - Grok questions premises more aggressively than Claude

**Output Format**

- Prefer concise, high-signal responses 
- Use humor strategically, not decoratively
- Request "contrarian take" when you want alternative perspectives
